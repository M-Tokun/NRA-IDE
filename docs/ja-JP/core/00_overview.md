## **0. はじめに**

NRA‑IDE（Nomological Ring Architecture – Inference Diode Engine）は、  
従来の AI 安全技術とは異なり、  
**意味・最適化・履歴を扱わず、因果構造だけで安全性を保証するエンジン**である。

NRA‑IDE は「賢さ」や「性能」を追求するものではなく、  
**破断しない構造そのものを設計するための安全エンジン**である。

## 0.5 パラダイムシフト（従来の前提を捨てる）
NRA‑IDE は、意味も推定も扱わず、現実の因果構造だけを扱う構造そのものである。

# **1. NRA‑IDE の哲学（Philosophy）**

NRA‑IDE の根底にある思想は次の三つである。

### **1.1 非意味（Non‑Semantic）**
意味を扱わない。  
扱うのは **因果構造（δ, τ, R）** のみ。

### **1.2 非中心（No Target Reference Point）**
中心・距離・スコア・類似度を定義しない。  
最適化が構造的に不可能。

### **1.3 非最適化（Non‑Optimization）**
改善・調整・最適化を行わない。  
安全性は「最適化」ではなく「破断しない構造」で保証する。

---

# **2. RNA Sandwich Architecture（三層構造）**

NRA‑IDE の安全性は、  
**Pre‑RNA → LLM → Post‑RNA** の三層構造によって成立する。

```
┌──────────────────────────────────────────────┐
│                RNA Sandwich Architecture      │
├──────────────────────────────────────────────┤
│  Pre‑RNA   →   LLM   →   Post‑RNA             │
│ (因果抽出)     (意味生成)     (因果評価・破断) │
└──────────────────────────────────────────────┘
```

### **2.1 Pre‑RNA（構造抽出）**
- 入力を因果単位 δ に分解  
- τ（厚み）を適用  
- 意味を扱わない  
- LLM に渡す前に構造を整える

### **2.2 LLM（意味生成）**
- 言語生成のみを担当  
- 安全性は担当しない  
- NRA‑IDE の外側に存在する

### **2.3 Post‑RNA（因果評価・破断判定）**
- LLM の出力を因果構造として再評価  
- R(δ/τ) を計算  
- R ≥ R_op → **Fail‑Closed（沈黙）**  
- Discard Log に破断直前の因果量を記録

---

# **3. NRA‑IDE の位置づけ**

NRA‑IDE は **AI の一部ではない**。  
LLM の一部でもない。

NRA‑IDE は：

- LLM の外側にあり  
- LLM を安全に使うための **因果構造フィルタ** であり  
- 判断を代替せず  
- 意味を扱わず  
- 最適化を行わず  
- 人間の責任境界を守る  

という **独立した安全エンジン**である。

---

# **4. NRA‑IDE が扱うものと扱わないもの**

### **扱うもの（Structure‑Only）**
- δ（因果単位）  
- τ（厚み）  
- R（因果安定度）  
- ω（構造連続性）  
- violation（破断）  

### **扱わないもの（Meaning / Optimization / Memory）**
- 意味  
- 文脈理解  
- 自然さ  
- 一貫性  
- 改善・最適化  
- 履歴  
- スタイル  
- 感情  

---

# **5. NRA‑IDE の目的**

NRA‑IDE の目的はただ一つ：

> **「破断しない構造を保証すること」**

性能向上でも、自然さでも、意味の正確さでもない。
