
# **12_appendix.md — 付録（Appendix）**

## **0. はじめに**

本付録は、NRA‑IDE の理解に必要な  
**補助的な定義・記号・構造要素**をまとめたものである。

ここでは、  
- 用途  
- 応用例  
- 産業名  
- 運用判断  

などには一切踏み込まない。

NRA‑IDE の構造体系を支えるための  
**最小限の技術的補助情報**のみを扱う。

---

# **1. 記号一覧（Symbols）**

NRA‑IDE で使用する主要な記号をまとめる。

### **δ（Delta）**  
入力を構造的に分解した最小単位。  
意味を含まない。

### **τ（Tau）**  
構造の「厚み」。  
入力 δ に適用される構造的フィルタ。

### **R（Risk / Ratio）**  
因果構造の安定度を示す量。  
R = f(δ/τ)

### **R_op（Operational Threshold）**  
運用上の閾値。  
R ≥ R_op で Fail‑Closed（沈黙）。

### **Discard Log**  
破断直前の因果量のみを記録するログ。  
意味を含まない。

---

# **2. 用語定義（Glossary）**

### **Fail‑Closed**  
不確実なら沈黙する安全原則。  
修正・補完・推測は行わない。

### **Symbol‑Only Principle**  
意味を扱わず、記号（構造）だけを扱う原則。

### **Curtain**  
Pre‑RNA / LLM / Post‑RNA の分離が薄くなり、  
逆流が発生する危険状態。

### **逆流（Backflow）**  
LLM の意味生成が RNA の因果構造に干渉する現象。

### **構造侵食（Structure Erosion）**  
意味・最適化・履歴が構造層に侵入し、  
因果構造が破壊される現象。

### **意味の染み出し（Semantic Leakage）**  
意味が構造層に侵入し、  
因果評価が意味評価に書き換わる最も危険な破断。

### **計算的再構成（Calculated Reconstruction）**  
意味ではなく因果構造（δ, τ, R）に基づき、  
推定ではなく **計算** によって構造を再構成する手法。

---

# **3. NRA‑IDE の三層構造（Three‑Layer Architecture）**

NRA‑IDE は以下の三層で構成される。

### **3.1 Pre‑RNA**  
- 入力 δ の抽出  
- τ の適用  
- 意味解析は行わない  

### **3.2 LLM**  
- 言語生成のみ  
- 安全性を担当しない  
- RNA の判断を参照しない  

### **3.3 Post‑RNA**  
- R の計算  
- R ≥ R_op で沈黙  
- Discard Log の記録  

---

# **4. NRA‑IDE の動作フロー（Operational Flow）**

```
入力 → Pre‑RNA（δ抽出）  
      → LLM（生成）  
      → Post‑RNA（R計算）  
      → R < R_op → 出力  
      → R ≥ R_op → Fail‑Closed（沈黙）
```
