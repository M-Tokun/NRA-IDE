
# **01_paradigm_shift.md — パラダイムシフト：NRA‑IDE がもたらす構造転換**

## 🧭 はじめに：NRA‑IDE は“改良版ガードレール”ではない

NRA‑IDE は、従来の AI 安全技術の延長線上にあるものではありません。  
これは **「最適化ベースのAI」から「因果構造ベースのAI」への転換**を意味します。

従来のアプローチは、  
- スコア  
- 距離  
- 目標  
- 報酬  
- フィルタ  
- ガードレール  

といった **Effect（射影）側の情報**を使って安全性を確保しようとします。

しかし NRA‑IDE は、  
**Cause（因果）側にのみアクセスし、Effect を完全に切り離す**  
という構造を採用します。

これが **パラダイムシフト**です。

---

# 🎯 1. 従来のAI安全の限界

従来の安全対策は、  
「出力されたテキストを見て危険ならブロックする」  
という **事後フィルタ方式**が中心でした。

しかしこの方式には構造的な限界があります：

### ❌ 1. 幻覚を防げない  
出力が生成された後に判定するため、  
**生成プロセスそのものは暴走可能**。

### ❌ 2. Goodhart の法則に弱い  
スコアや距離を使うと、  
モデルは **スコアを最大化する方向に“ゲーム化”** する。

### ❌ 3. ブラックボックス性が増す  
内部で何が起きているか観測できない。

### ❌ 4. モデル依存  
モデルごとに安全対策を作り直す必要がある。

---

# 🎯 2. NRA‑IDE の核心：因果構造による制御

NRA‑IDE は、  
**「出力を安全にする」のではなく、  
　“因果構造の破断を防ぐ”** という発想に基づきます。

そのために以下の構造を採用します：

### ✔ 逆導出（Π⁻¹）の禁止  
Effect → Cause の逆算を完全に遮断。

### ✔ 距離・スコア・座標を使わない  
中心が存在しないため、距離は定義できない。

### ✔ 境界（Constraint）だけが存在する  
中心（Target）は存在しない。

### ✔ Fail‑Closed（不確実なら沈黙）  
破断時は沈黙し、回復試行は行わない。

### ✔ 熱廃棄ログによる透明性  
捨てられた経路を記録し、LLM は改ざんできない。

---

# 🎯 3. 「中心がない」という構造的決断

従来のAIは「中心（目標）」を前提にしています。

```
中心 → 距離 → 最適化
```

しかし NRA‑IDE は **中心を持たない構造**を採用します。

```
中心がない → 距離が定義できない → 最適化が不可能
```

存在するのは **境界（Constraint）** のみ。

境界には：

- 厚み（τ）  
- ゆらぎ（δ）  
- 動的変動  

があり、  
**「内側か外側か」だけが判定される**。

---

# 🎯 4. 最適化からの脱却

従来のAIは「最適化」を前提にしています。

- 距離を最小化  
- スコアを最大化  
- 報酬を最大化  

しかし最適化は必ず **Goodhart の法則** を引き起こします。

NRA‑IDE は最適化を完全に排除し、  
**構造的不変条件（Invariant）** のみを扱います。

---

# 🎯 5. 「意味」を扱わないという決断

NRA‑IDE は **意味（Semantic）を扱いません**。

理由：

- 意味は射影（Effect）であり因果ではない  
- 意味評価は逆導出（Π⁻¹）を誘発する  
- 意味はモデル依存であり安全性が揺らぐ  

代わりに扱うのは：

- ゆらぎ（δ）  
- 厚み（τ）  
- 角速度（ω）  
- 逸脱量（violation）  

といった **因果的に定義可能な量**のみ。

---

# 🎯 6. Fail‑Closed（不確実なら沈黙）という構造的決断

NRA‑IDE では、  
**破断が検出された場合は沈黙する（Fail‑Closed）** という  
構造的不変条件を採用しています。

Fail‑Closed は以下を意味します：

- 不確実な状態では出力しない  
- 構造逸脱が検出された時点で即時停止  
- 回復試行（再生成）は行わない  
- 沈黙は「安全性の証明」である  

さらに重要なのは：

> **Fail‑Closed は適切に設定された閾値（厚み τ）とゆらぎ（δ）の調整を前提とする。  
> 誤設定は過剰沈黙または過剰発話を引き起こす可能性がある。**

Fail‑Closed は「機能」ではなく、  
**因果構造の破断を防ぐための設計原理**です。

---

# 🎯 7. NRA‑IDE がもたらすもの

NRA‑IDE は以下を実現します：

- **構造的に幻覚を防ぐ**  
- **最適化による暴走を防ぐ**  
- **モデル非依存の安全性**  
- **透明性の高いログ（熱廃棄ログ）**  
- **因果構造の破断を検出できる**  
- **安全エージェントの基盤になる**

これは単なる技術ではなく、  
**AI の扱い方そのものを変える思想**です。
