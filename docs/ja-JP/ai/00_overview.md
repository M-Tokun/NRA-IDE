
## 0. この文書の目的

本書は NRA‑IDE（Nomological Ring Axioms – Intensional Dynamics Engine）の  
**最初の入口**であり、以降の章を読むための  
「前提」「禁止事項」「構造的姿勢」を明確にする。

NRA‑IDE は従来の AI 安全技術の延長ではなく、  
**前提そのものを捨てる必要があるパラダイムシフト**である。

そのため、本書ではまず  
**“やってはいけないこと（禁止事項）”** を明確にし、  
誤った理解のまま進むことを防ぐ。

詳細な説明は後続の章で行う。

---

## 0.5 パラダイムシフト（従来の前提を捨てる）
NRA‑IDE は、意味も推定も扱わず、現実の因果構造だけを扱う構造そのものである。
証明も実験も不要で、やればわかる。必要なのは厳密な調整だけである。
NRA‑IDE を理解するには、  
**　既存の前提（距離・座標・意味・最適化）を一度リセットする必要がある。**

従来の AI 安全は：

- 距離  
- スコア  
- 類似度  
- 意味解析  
- 最適化  
- ガードレール  
- RLHF  

といった **Effect（射影）** を前提にしていた。

しかし NRA‑IDE は：

- 中心を持たない  
- 距離が定義できない  
- 意味を扱わない  
- 最適化を行わない  
- 逆導出（Π⁻¹）を禁止する  

という **完全に異なる前提**で動作する。

---

## 0.6 （重要）禁止事項の概要  
※ 詳細は **13_warning.md** を参照

NRA‑IDE では、以下の入力・操作・判断は  
**構造破断（Fracture）を引き起こすため絶対に禁止される。**

### ❌ 禁止されるもの（Effect / Semantic）

- 単語・熟語・文  
- 意味（Semantic）  
- 評価語（危険、安全、正しい、間違い など）  
- 距離・スコア・類似度  
- 過去出力  
- “改善”という行為（最適化）  
- 意味ログを学習に使うこと  
- Discard Log を LLM に戻すこと  

これらはすべて **Effect（射影）** であり、  
LLM に渡すと **逆導出（Effect → Cause）** を誘発し、  
NRA‑IDE の構造安全性を破壊する。

### ✔ 許可されるもの（Cause / Structure）

- 記号（Symbol）  
  - [], {}, (), <>, =, +, -, *, / など  
- 数値  
- 因果量（δ, τ, R, ω, violation, workRate）  
- 構造操作子（Operators）

**LLM に渡してよいのは記号のみ。**  
詳細は **13_warning.md** にて定義する。

## 1. NRA‑IDE の概要（構造の入口）

NRA‑IDE は、大規模言語モデルに対して  
**因果構造レベルの安全性を外部から付与するエンジン**である。

従来の「意味ベース安全性」とは異なり、  
NRA‑IDE は **因果構造のみ** を扱う。

これにより：

- 幻覚（Hallucination）の構造的抑制  
- Fail‑Closed（不確実なら沈黙）  
- 逆導出（Π⁻¹）の完全禁止  
- 熱廃棄ログによる透明性  
- モデル非依存の安全性  

が実現される。

---

## 2. 基本構造：RNA サンドイッチ構造

NRA‑IDE は以下の三層構造で動作する：

```
INPUT
  ↓
[ Pre‑RNA Box：因果制約の注入 ]
  ↓
[ LLM（Plant）：生成器 ]
  ↓
[ Post‑RNA Box：因果監査・破断判定 ]
  ↓
OUTPUT
```

### Pre‑RNA  
- 意味を扱わない  
- 距離・スコアを排除  
- 因果量のみを渡す  

### LLM（Plant）  
- 内部状態はブラックボックス  
- 意味の整合性は LLM に任せる  
- 安全性は LLM に任せない  

### Post‑RNA  
- 意味評価を行わない  
- 構造逸脱のみを監査  
- 破断時は Fail‑Closed  

---

## 3. NRA‑IDE が解決する問題（構造的視点）

### ✔ 幻覚（Hallucination）  
意味ではなく **構造破断** として検出し沈黙する。

### ✔ ブラックボックス性の低減  
Discard Log により、破断直前の因果量が観測可能。

### ✔ モデル依存性の排除  
意味を扱わないため、どの LLM にも適用可能。

## 4. 運用上の注意（必読）

- RNA サンドイッチ構造なしでの使用は禁止  
- 閾値（δ / τ）の誤設定は危険  
- 意味評価は構造破断の原因  
- 逆導出（Π⁻¹）は公理違反  
- Fail‑Closed はエラーではなく仕様
