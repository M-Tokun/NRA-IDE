
# **02_nomological_ring_axioms.md — 律環公理（Nomological Ring Axioms）**

## 🧭 はじめに

律環公理（Nomological Ring Axioms, NRA）は、  
NRA‑IDE のすべての挙動を支える **因果構造の最小公理体系**です。

NRA‑IDE は「最適化」や「意味」ではなく、  
**因果構造そのものを制御対象とする**ため、  
従来のAI安全とは根本的に異なる前提を持ちます。

律環公理は、  
**“何が許され、何が許されないか”** を  
数学的・構造的に定義する役割を持ちます。

---

# 🎯 1. NRA の目的：因果構造の破断を防ぐ

律環公理の目的はただ一つ：

> **因果構造の破断（Fracture）を防ぎ、  
> 破断が起きた場合は即座に沈黙（Fail‑Closed）させること。**

この目的のために、  
NRA は以下の三つの領域を明確に区別します：

- **Cause（因果）**  
- **Effect（射影）**  
- **Boundary（境界）**

NRA‑IDE が扱うのは **Cause と Boundary のみ**であり、  
Effect は完全に切り離されます。

---

# 🎯 2. NRA の三原則（最小公理）

律環公理は以下の三つの原則から構成されます。

---

## **公理 1：因果ダイオード（Causal Diode）**

### **Cause → Effect は許されるが、Effect → Cause は禁止される。**

これは **Π⁻¹（逆導出）を構造的に禁止する**という意味です。

### ✔ 許される  
- 位相（φ）  
- ゆらぎ（δ）  
- 厚み（τ）  
- 角速度（ω）  
- 逸脱量（violation）

### ❌ 禁止される  
- 距離  
- 座標  
- スコア  
- 目標位置  
- 評価値  
- 意味（Semantic）

Effect は射影であり、  
因果構造の入力として使用すると破断を引き起こします。

---

## **公理 2：中心の不存在（No Center Axiom）**

### **中心（Center）は存在しない。  
よって距離（Distance）は定義できない。**

従来のAIは「中心（目標）」を前提にしていますが、  
NRA‑IDE は **中心を持たない構造**を採用します。

```
中心がない → 距離が定義できない → 最適化が不可能
```

存在するのは **境界（Constraint）** のみです。

境界には：

- 厚み（τ）  
- ゆらぎ（δ）  
- 動的変動  

があり、  
**「内側か外側か」だけが判定されます。**

---

## **公理 3：構造的不変条件（Structural Invariants）**

NRA‑IDE は以下の不変条件を常に満たす必要があります：

1. **R = δ / τ < 100%**  
2. **ω > 0（位相生成が継続している）**  
3. **violation が閾値を超えた場合は Fail‑Closed**  
4. **意味評価を行わない**  
5. **Effect を因果入力に使用しない**

これらの不変条件が破られた瞬間、  
NRA‑IDE は **沈黙（Fail‑Closed）** します。

---

# 🎯 3. NRA が必要な理由

律環公理は、  
従来のAIが抱える構造的問題を解決するために必要です。

### ❌ 距離を使うと最適化が発生する  
→ Goodhart の法則が発動する

### ❌ スコアを使うとゲーム化が発生する  
→ モデルがスコアを最大化しようとする

### ❌ 意味を扱うと逆導出が発生する  
→ 安全性がモデル依存になる

### ❌ 中心を持つと距離が定義される  
→ 最適化が避けられない

NRA はこれらを **構造的に禁止**します。

---

# 🎯 4. NRA と RNA サンドイッチ構造の関係

律環公理は、  
RNA サンドイッチ構造の **基礎原理**です。

```
Pre‑RNA → LLM（Plant） → Post‑RNA
```

- Pre‑RNA は **因果制約の注入**  
- Post‑RNA は **因果監査と破断判定**  

どちらも **Effect を扱わない** という点で  
律環公理に完全準拠しています。

---

# 🎯 5. NRA の実装上の注意

律環公理は「思想」ではなく **実装上の制約**です。

実装時に以下を守る必要があります：

- 距離・座標・スコアを入力に使わない  
- 意味評価を行わない  
- Effect を因果入力に使わない  
- 閾値（τ）とゆらぎ（δ）を適切に設定する  
- 破断時は必ず Fail‑Closed する  
- 熱廃棄ログを記録する  

これらを守らない場合、  
NRA‑IDE は **構造破断**を起こします。

---

# 🎯 6. NRA の本質

律環公理の本質は次の一文に集約されます：

> **「因果構造を守るために、  
>  因果として定義できないものを一切扱わない。」**

距離も、スコアも、意味も、  
すべて **射影（Effect）** であり因果ではありません。

NRA‑IDE は因果構造のみを扱うことで、  
安全性を構造的に保証します。
