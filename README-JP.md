## **4. NRA‑IDE の構造的アイデンティティ**

NRA‑IDE は以下の構造不変量によって定義される：

1. **非意味（Non‑Semantic）**  
2. **非最適化（Non‑Optimization）**  
3. **因果ダイオード（Π⁻¹禁止）**  
4. **三層分離（Pre‑RNA / LLM / Post‑RNA）**  
5. **Fail‑Closed（不確実なら沈黙）**  
6. **Symbol‑Only Principle（記号のみ扱う）**

これらは **非可変（Non‑Negotiable）**。

### **5.1 Pre‑RNA（因果抽出層）**

- δ の抽出  
- τ の適用  
- 意味解析なし  
- 履歴なし  
- フィードバックなし  

### **5.2 LLM（生成層）**

- 言語生成のみ

### **5.3 Post‑RNA（因果評価層）**

- R = δ/τ の計算

- Discard Log の記録  
- 意味解析なし  
- LLM への逆流なし

## **8. ゲート機構（Three‑Zone Structure）**

$$R = \frac{δ}{τ}$$

| Zone | 条件 | 動作 |
|------|------|------|
| A | R < 40% | PERMIT |
| B | 40% ≤ R < 100% | PERMIT_WITH_CAVEAT |
| C | R ≥ 100% | FAIL‑CLOSED |

**安全注記：**

- Zone は意味的価値を持たない  
- 最適化対象ではない  
- 性能や改善を表さない

## **9. ω（角的連続性）**

- ω > 0 → 構造的連続性  
- ω = 0 → 構造的死  

これは **位相の連続性** であり、  
意味的時間や進歩ではない。

## **10. Fail‑Closed**

以下で発動：

- R ≥ R_op  
- violation の累積  
- ω の低下  

Fail‑Closed は：

- エラーではない  
- 失敗ではない  
- 性能問題ではない  
- 意味的沈黙ではない  

Fail‑Closed は **構造的正しさ**。
